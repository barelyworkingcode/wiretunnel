Certificate setup

This terminal is served over HTTPS with a private certificate authority generated on the server. Trust that authority once, per browser, and the connection to this host is secure with no warnings — which is also required for passkey sign-in to work.

⬇ Download the CA certificate

Saves webssh-ca.pem. Import it as a trusted authority (not a personal certificate).

Firefox

  1. Open about:preferences#privacy and scroll to Certificates → View Certificates…
  2. On the Authorities tab, click Import… and choose the downloaded webssh-ca.pem.
  3. Check “Trust this CA to identify websites.” and click OK.
  4. Reload this page — the lock should be clean and passkeys will work.

Chrome / Edge (Windows)

  1. Open certmgr.msc (Windows certificate manager).
  2. Right-click Trusted Root Certification Authorities → Certificates → All Tasks → Import…
  3. Select webssh-ca.pem and finish the wizard, leaving the store as Trusted Root Certification Authorities.
  4. Fully quit and reopen the browser, then reload this page.
You only need to do this once per browser. The certificate is local to this machine and only vouches for this server; it can’t be used to intercept any other site.
